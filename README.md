# 🛡️ LLM-SQL-Guard: A Defense Framework for Secure Text-to-SQL Integration

### **Enterprise-Grade Security Middleware for Generative AI & Data Infrastructure**

---

## 📖 Overview
**LLM-SQL-Guard** is a robust security middleware designed to bridge the gap between Large Language Models (LLMs) and enterprise relational databases. As businesses adopt **Copilot** and **Text-to-SQL** solutions, they face two critical challenges: **LLM Hallucinations** and **SQL Injection/Malicious Query Execution**.

This project provides a multi-layered defense mechanism—**SQL-Guard**—to intercept, analyze, and sanitize AI-generated SQL before it touches the production data layer.

---

## ✨ Key Features & Research Highlights

* **🛡️ Hallucination Mitigation**: Implements a semantic validation layer to ensure generated SQL aligns with the actual database schema, preventing "hallucinated" column names or tables.
* **🔒 Injection Defense**: A specialized parser that detects and blocks unauthorized DDL/DML operations (e.g., `DROP`, `TRUNCATE`, `UPDATE` without `WHERE`) generated by adversarial prompts.
* **🧩 Schema-Aware Mapping**: Leverages character-level mapping logic to translate generic AI-generated field names into legacy system column names (Ref: Insurance Core Systems / eBAF).
* **⚡ High-Performance Interceptor**: Designed with a low-latency architecture to ensure minimal overhead in high-concurrency environments (up to **10,000+ TPS**).
* **🤖 Framework Agnostic**: Seamlessly integrates with **Google Cloud Vertex AI**, **Azure OpenAI**, or local LLM deployments.

---

## 🏗️ Technical Architecture (The SA Perspective)

The framework operates on a **Zero-Trust** principle for AI-generated code:

1.  **Input Sanitization**: Cleans the Prompt and Schema Context.
2.  **Logic Correction Layer**: Rectifies common SQL syntax errors produced by LLMs.
3.  **Security Sandbox**: Performs a dry-run in a virtual schema to validate execution impact.
4.  **Final Execution**: Passes only verified, read-only (or authorized) queries to the database.
